## 四、数据库设计（MySQL 8.0.42）

> 字符集：`utf8mb4`，排序规则：`utf8mb4_unicode_ci`，引擎：InnoDB。

### 用户表 `users`

| 字段          | 类型            | 约束                          | 说明            |
| ------------- | --------------- | ----------------------------- | --------------- |
| id            | BIGINT UNSIGNED | PK, AUTO_INCREMENT            | 自增主键        |
| username      | VARCHAR(64)     | UNIQUE, NOT NULL              | 用户名          |
| password_hash | VARCHAR(256)    | NOT NULL                      | bcrypt 哈希密码 |
| created_at    | DATETIME        | DEFAULT NOW()                 | 注册时间        |
| updated_at    | DATETIME        | DEFAULT NOW() ON UPDATE NOW() | 更新时间        |

### 会话表 `sessions`

| 字段       | 类型            | 约束               | 说明                             |
| ---------- | --------------- | ------------------ | -------------------------------- |
| id         | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 自增主键                         |
| user_id    | BIGINT UNSIGNED | FK → users.id      | 关联用户                         |
| title      | VARCHAR(128)    | NOT NULL           | 会话标题（取第一条问题前 20 字） |
| created_at | DATETIME        | DEFAULT NOW()      | 创建时间                         |

### 消息表 `messages`

| 字段       | 类型            | 约束               | 说明                            |
| ---------- | --------------- | ------------------ | ------------------------------- |
| id         | BIGINT UNSIGNED | PK, AUTO_INCREMENT | 自增主键                        |
| session_id | BIGINT UNSIGNED | FK → sessions.id   | 关联会话                        |
| role       | VARCHAR(16)     | NOT NULL           | `user` / `assistant`            |
| content    | LONGTEXT        | NOT NULL           | 消息内容（Markdown）            |
| sources    | JSON            | NULL               | RAG 召回来源（MySQL JSON 类型） |
| created_at | DATETIME        | DEFAULT NOW()      | 发送时间                        |

### 建库 SQL

```sql
CREATE DATABASE IF NOT EXISTS scu_qa
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE scu_qa;

CREATE TABLE users (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  username      VARCHAR(64)  NOT NULL UNIQUE,
  password_hash VARCHAR(256) NOT NULL,
  created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE sessions (
  id         BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id    BIGINT UNSIGNED NOT NULL,
  title      VARCHAR(128) NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE messages (
  id         BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  session_id BIGINT UNSIGNED NOT NULL,
  role       VARCHAR(16) NOT NULL,
  content    LONGTEXT NOT NULL,
  sources    JSON,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 五、SQLAlchemy 连接配置（MySQL 异步）

```python
# backend/models/database.py
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession
from sqlalchemy.orm import DeclarativeBase, sessionmaker

# 异步驱动：aiomysql
DATABASE_URL = "mysql+aiomysql://root:password@localhost:3306/scu_qa"

engine = create_async_engine(DATABASE_URL, echo=False, pool_size=10)
AsyncSessionLocal = sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)

class Base(DeclarativeBase):
    pass

async def get_db():
    async with AsyncSessionLocal() as session:
        yield session
```

## 七、用户认证流程

```
注册：POST /api/auth/register
  Body: { username, password }
  → passlib bcrypt 哈希密码
  → 写入 MySQL users 表
  → 返回 { id, username }

登录：POST /api/auth/login
  Body: { username, password }
  → 查询用户 → 验证密码哈希
  → 签发 JWT（有效期 7 天）
  → 返回 { access_token, token_type: "bearer" }

前端：
  JWT 存入 localStorage
  Axios 拦截器自动附加 Authorization: Bearer <token>
  路由守卫：未登录自动跳转 /login

受保护接口：
  Depends(get_current_user) → 解析 JWT → 返回当前用户对象
```

## 八、RAG 流程设计

```
【离线阶段 — 知识库构建，运行 ingest.py】

原始文档（PDF/TXT/MD）
    │
    ▼
文档解析（LangChain DocumentLoader：PyPDFLoader / TextLoader）
    │
    ▼
文本分块（RecursiveCharacterTextSplitter，chunk_size=512，overlap=64）
    │
    ▼
Embedding（bge-large-zh，本地运行，无 API 费用）
    │
    ▼
存入 ChromaDB（persist_directory="./backend/chroma_db"）

【在线阶段 — 用户提问】

用户问题
    │
    ▼
问题 Embedding → ChromaDB 相似度检索（Top-K = 5）
    │
    ▼
召回文档片段 + 来源信息
    │
    ▼
Prompt 拼装：
┌──────────────────────────────────────────────────────┐
│ system: 你是四川大学智能助手，请依据参考资料用中文回答。  │
│         若资料不足，请如实说明，不要编造。              │
│ user:   参考资料：{context}                           │
│         问题：{question}                              │
└──────────────────────────────────────────────────────┘
    │
    ▼
DeepSeek API 流式调用（stream=True）
    │
    ▼
SSE 推送至前端（打字机效果）+ 返回来源引用
```

## 九、导航功能流程

```
用户输入："从望江楼到化工学院怎么走"
    │
    ▼
后端意图识别（正则优先，fallback LLM）：
  type: "navigation"，origin: "望江楼"，destination: "化工学院"
    │
    ▼
查询 landmarks.json 获取坐标：
  origin_lnglat: [104.0827, 30.6297]
  dest_lnglat:   [104.0851, 30.6312]
    │
    ▼
调用高德步行路径规划 REST API（后端 httpx）：
  GET https://restapi.amap.com/v3/direction/walking
    │
    ▼
解析返回数据（steps 文字 + polyline 坐标串）
    │
    ▼
返回前端 JSON：
{
  "origin": "望江楼", "destination": "化工学院",
  "steps": ["沿xxx路向北走100米", ...],
  "polyline": "104.0827,30.6297;...",
  "distance": 650, "duration": 480
}
    │
    ▼
前端高德 JS API 2.0 绘制路线 + 起终点 Marker
```

## 十六、关键技术要点

1. **流式输出（SSE）**：FastAPI 使用 `StreamingResponse`，前端用 `fetch` + `ReadableStream` 逐字追加，实现打字机效果。
2. **MySQL JSON 字段**：`messages.sources` 使用 MySQL 8.0 原生 JSON 类型，直接存储 RAG 来源数组，无需序列化。
3. **异步数据库**：全链路异步（`aiomysql` + SQLAlchemy async），避免 I/O 阻塞，FastAPI 性能最大化。
4. **Alembic 迁移**：所有表结构变更通过 Alembic 管理，禁止直接 `ALTER TABLE`，保持版本可追溯。
5. **导航意图识别**：正则优先（含「怎么走」「路线」「在哪」等关键词），未命中再 fallback 到 LLM 提取实体，降低延迟。
6. **向量库冷启动**：`ingest.py` 首次运行耗时，ChromaDB 持久化后后续启动无需重建。
7. **高德 JS Key 安全**：在高德控制台绑定 `referer` 白名单（开发填 `localhost:5173`）。
8. **密码安全**：`passlib[bcrypt]` 哈希存储，cost factor 默认 12，禁止明文。
9. **Python 3.14 兼容**：部分 C 扩展库可能尚未发布 3.14 wheel，安装时如遇问题可先在虚拟环境用 `--pre` 或源码安装。
