## 1 引言

### 1.1 编写目的

本项目文档旨在为基于大模型与RAG技术的四川大学校内问题解答网站项目提供全面的项目管理和实施指导。文档明确了项目的背景、目标、需求、技术方案、团队组织、实施计划以及相关支持条件，为项目团队和相关利益方提供参考依据，确保项目顺利实施并达到预期目标。

### 1.2 背景

随着人工智能技术的快速发展，大语言模型在教育领域的应用日益广泛。四川大学作为国内知名高校，校园内信息量庞大，学生和教职工对智能问答系统的需求日益迫切。本项目旨在构建一个基于大模型和检索增强生成（RAG）技术的校内问题解答网站，为用户提供准确、实时的校园信息查询服务，包括课程安排、校园设施、活动通知等。

项目采用前端Vue.js + 后端FastAPI的技术栈，集成DeepSeek大模型API和高德地图导航功能，实现本地化部署和数据安全。

### 1.3 定义

- **RAG（Retrieval-Augmented Generation）**：检索增强生成技术，通过检索相关知识库信息来增强大模型的回答准确性。
- **大模型**：指DeepSeek等大型语言模型，用于生成自然语言回答。
- **前端**：用户界面层，使用Vue.js框架构建。
- **后端**：服务端逻辑层，使用FastAPI框架构建。
- **向量数据库**：用于存储和检索文本嵌入向量的数据库，如ChromaDB。
- **JWT**：JSON Web Token，用于用户身份验证。

## 2 项目概述

### 2.1 项目目标

构建一个智能、高效、安全的四川大学校内问题解答网站，为师生提供基于大模型的智能问答服务，实现校园信息的精准检索和个性化回答，提升用户体验和信息获取效率。

### 2.2 项目详细需求内容

- **用户管理**：支持用户注册、登录、个人信息管理。
- **智能问答**：基于RAG技术，提供校园相关问题的智能回答。
- **地图导航**：集成高德地图，实现校内路线规划和设施查询。
- **聊天界面**：提供友好的聊天界面，支持Markdown渲染。
- **数据安全**：确保用户数据和聊天记录的安全存储。
- **响应式设计**：支持PC和移动端访问。

### 2.3 关键技术

前端技术：Vue 3, Vite, Vue Router, Pinia, Element Plus, Axios  
后端技术：Python 3.14, FastAPI, LangChain, ChromaDB, MySQL, SQLAlchemy, JWT  
AI技术：DeepSeek API, Sentence-Transformers, RAG架构  
部署技术：Uvicorn服务器

### 2.4 项目资源与环境要求

硬件要求：服务器配置至少4核CPU、8GB内存、100GB存储  
软件环境：Windows操作系统，Python 3.14, Node.js 18+, MySQL 8.0  
开发工具：VS Code, Git, Postman  
网络要求：稳定的互联网连接，支持API调用
